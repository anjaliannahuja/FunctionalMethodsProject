Basic Events with jQuery

Events are messages that occur when you interact with elements on the DOM. For instance, any element you click on with your mouse will send (or emit) a click event. Ordinarily, clicking on most elements has no effect, but you can assign an effect using jQuery's .on() method. The first argument passed to it should be the string 'click', and the second one should be a callback function. It looks like this:

1
<div id="clickable">Click me to see a console.log()</div>

1
2
3
4
// "e" here is the object representing the event
$('#clickable').on('click', function(e) {
    console.log('Hello!');
});

If you then click on that <div> with the id="clickable", even though it is not a button, it will log a message. What we've done here is added a function that waits for an event to happen before executing, called an event listener. You can assign multiple event handlers to the same element:

});

Now if you were to click on the element, you would see "Hello!" three times, "Hi!" once, and "Bye!" once in your console.

Since we can pass a string to .on(), this seems to suggest that there are other types of events too. There are! There are a whole bunch, as a matter of fact:

  • 'blur' - this event is emitted when an element loses focus, which happens when the user has focus on an element and Tabs to another element or clicks elsewhere
  • 'change' - this event is emitted when the value of an input field is changed, such as a text area, text input field, radio button, etc.
  • 'click' - this event is emitted when an element is clicked on
  • 'dblclick' - this event is emitted when an element is double-clicked on
  • 'error' - this event is emitted when an element is not loaded correctly, such as <img> elements
  • 'focus' - this event is emitted when an element gains focus, meaning that the user has reached the element via the Tab key or has clicked on it
  • 'hover' - this event is emitted when the mouse pointer enters or leaves the element
  • 'keydown' - this event is emitted when the user first presses any key down
  • 'keyup' - this event is emitted when the user releases a key
  • 'mousedown' - this event is emitted when the user presses a mouse button over the targeted element
  • 'mouseup' - this event is emitted when the user releases a mouse button over the target element
  • 'mouseenter' - this event is emitted when the user moves the mouse pointer into the targeted element
  • 'mouseleave' - this event is emitted when the user moves the mouse pointer out of the targeted element
  • 'mousemove' - this event is emitted when the user moves the mouse around within the targeted element
  • 'ready' - this event is emitted when the DOM is fully loaded; generally only use with the document element
  • 'resize' - this event is emitted when the targeted element is resized
  • 'scroll' - this event is emitted when the target element is scrolled
  • 'select' - this event is emitted when the target element is selected, usually only used for various types of inputs
  • 'submit' - this event is emitted when the target element is submitted, and can only be attached to form inputs

As you can see, you can attach an event listener to practically every user interaction possible besides thought (for now). There are even a whole slew of touch-related events for mobile devices and tablets which are outside the scope of this lesson. You're encouraged to toy around with all of the different events and see what's possible!

Getting the Targeted Element

Let's say that we want to be able to click on an element and get arbitrary data from it in the form of a data attribute (discussed in Changing Element Attributes). However, how do we get the actual element that was clicked on? We do this using the this keyword.

In jQuery, whenever you use .on(), the value of this in your event listener function will refer to the element being targeted by that event listener. For example:

1
<div id="clickable" data-id="5">A clickable div</div>

1
2
3
4
5
$('#clickable').on('click', function() {
    var jQueryElement = $(this);
    var val = jQueryElement.data('id');
    console.log(val); // logs 5
});

You'll notice that we had to wrap this in the jQuery function $(). The reason for this is that jQuery sets this to point to the actual DOM element we triggered the event handler on, not a jQuery object wrapping that DOM element. Therefore, if we want to use jQuery methods on this, we need to first wrap it in a jQuery object.

Once we've done this, we can access the elements with the expressive power of jQuery.
