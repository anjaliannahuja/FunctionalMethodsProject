jQuery - A Conceptual Overview

jQuery is a JavaScript library designed to make it easier to manipulate a web page.

This includes:

  • selecting elements
  • adding/removing classes and attributes from elements
  • hiding/showing/deleting/moving elements
  • adding events to elements
  • ...and much, much, more

How is jQuery Easier?

Consider the following example JavaScript code that gets an element by its ID and then removes it from the DOM:

1
2
var el = document.getElementById('element-id');
el.parentNode.removeChild(el);

Here's that same code written using jQuery:

1
$('#element-id').remove();

jQuery also normalizes a lot of subtle differences in how older browsers implement a variety of JavaScript functionalities. This means that if you use jQuery, you can use the same methods to manipulate and access the DOM in almost all browsers.

In general, jQuery makes life a lot easier all around. This set of lessons will teach you all the fundamentals of how to get started using it.

The $ Character

In JavaScript, the $ character is a completely valid character to use in variable names. jQuery uses the $ character as a shorthand. You can use it interchangeably with jQuery as a way to reference the jQuery library:

1
2
3
// Both functionally the same
$('#element-id').remove();
jQuery('#element-id').remove();

While we're on the topic, how do jQuery() and $() work? $() looks especially weird. What's happening here is that $() is a function call, meaning that $ is a function. This function is designed to take in a string (such as $('#element-id')), which will select an element. After this, we then call remove on that element. The $ variable is the entry point into all jQuery-related functionality.

jQuery Objects vs DOM Elements

You'll hear two terms thrown around a lot in this lesson--**DOM elements** and jQuery objects. This nomenclature can be a little confusing, but it's important to understand the distinction. DOM elements are the actual elements that we ordinarily grab with native JavaScript and then manipulate using the browser's DOM API. They're the browser-world representation of the HTML that we put in. jQuery objects "wrap" DOM elements in new objects that define properties and method names that are easier for people to use, like append and remove, instead of more verbose JavaScript operations like element.innerHtml = element.innerHtml + '<div>my new dom stuff</div>'. In addition, jQuery objects can wrap more than one element at a time, meaning that you can run many jQuery methods on multiple elements at the same time.

How do jQuery objects "wrap" DOM elements? This is just terminology that means that jQuery objects store references to the DOM elements they "wrap". When we call methods on jQuery objects, jQuery then goes and runs the native JavaScript methods on the actual DOM elements that the jQuery object is wrapping. This abstraction gives us a lot of power and flexibility in our manipulation of the DOM.
